# Aureus Website Redesign — Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task. Each task is self-contained and includes exact file paths, complete code, and verification commands.

**Goal:** Redesign the entire portfolio-tracker web app from the current dark "Console" theme to the "Aureus" design system — warm light/dark themes with serif display fonts, a fixed labelled sidebar, animated page transitions, slide-over drawers, a visible ⌘K command palette, and a new marketing landing page — using the HTML prototypes in `Financial tracker redesign/`.

**Architecture:** Swap the CSS design tokens in `globals.css` to the Aureus dual-theme (Ledger light / Onyx dark) palette. Replace the slim hover-expand sidebar rail with a fixed 250px labelled sidebar. Add a visible search field + ⌘K hint in the sidebar. Enhance the existing command palette styling. Add slide-over drawer + toast infrastructure. Add animated route transitions (sectionIn keyframe). Build a new marketing landing page as a public route. Copy the Aureus brand assets (face-ink.png, face-gold.png, roman-1.png) into `public/`.

**Tech Stack:** Next.js 13.5 (app router), React 18, Tailwind CSS 3, next-themes, sonner (toast), Radix UI Dialog, lucide-react, recharts. Fonts: Newsreader/Spectral (display serif), Hanken Grotesk (UI), IBM Plex Mono (mono labels).

```tsx
'use client'
import { useEffect, useMemo, useState } from 'react'
import { useTheme } from 'next-themes'
import { Moon, Sun, CornerDownLeft, Search } from 'lucide-react'
import { Dialog, DialogContent, DialogTitle } from '@/components/ui/dialog'
import { NAV_ROUTES } from '@/lib/nav-registry'
import { fuzzyScore } from '@/lib/fuzzy'
import { useViewTransitionRouter } from '@/components/motion/ViewTransitionProvider'
import { cn } from '@/lib/utils'

interface Item {
  key: string
  label: string
  hint: string
  icon: React.ElementType
  run: () => void
}

export function CommandPalette({ open, onOpenChange }: {
  open: boolean
  onOpenChange: (v: boolean) => void
}) {
  const [q, setQ] = useState('')
  const [active, setActive] = useState(0)
  const navigate = useViewTransitionRouter()
  const { theme, setTheme } = useTheme()

  const items: Item[] = useMemo(() => {
    const routes: Item[] = NAV_ROUTES.map((r) => ({
      key: r.href, label: r.label, hint: r.group, icon: r.icon,
      run: () => navigate(r.href),
    }))
    const actions: Item[] = [{
      key: 'toggle-theme',
      label: theme === 'light' ? 'Switch to dark' : 'Switch to light',
      hint: 'Theme', icon: theme === 'light' ? Moon : Sun,
      run: () => setTheme(theme === 'light' ? 'dark' : 'light'),
    }]
    return [...routes, ...actions]
  }, [navigate, theme, setTheme])

  const results = useMemo(() => {
    return items
      .map((it) => ({ it, s: fuzzyScore(it.label, q) }))
      .filter((x) => x.s !== null)
      .sort((a, b) => b.s! - a.s!)
      .map((x) => x.it)
  }, [items, q])

  useEffect(() => { if (open) { setQ(''); setActive(0) } }, [open])
  useEffect(() => { setActive(0) }, [q])

  const choose = (it?: Item) => { if (!it) return; onOpenChange(false); it.run() }

  return (
    <Dialog open={open} onOpenChange={onOpenChange}>
      <DialogContent className="max-w-[560px] gap-0 overflow-hidden rounded-2xl border border-border p-0 shadow-2xl animate-pop-in" aria-describedby={undefined}>
        <DialogTitle className="sr-only">Command palette</DialogTitle>
        <div className="flex items-center gap-3 border-b border-[var(--hair)] px-5 py-4">
          <Search className="h-4 w-4 text-faint" />
          <input
            autoFocus
            value={q}
            onChange={(e) => setQ(e.target.value)}
            placeholder="Search pages and actions…"
            className="flex-1 border-none bg-transparent text-[15px] outline-none placeholder:text-faint"
            onKeyDown={(e) => {
              if (e.key === 'ArrowDown') { e.preventDefault(); setActive((v) => Math.min(results.length - 1, v + 1)) }
              else if (e.key === 'ArrowUp') { e.preventDefault(); setActive((v) => Math.max(0, v - 1)) }
              else if (e.key === 'Enter') { e.preventDefault(); choose(results[active]) }
            }}
          />
          <span className="font-mono rounded border border-border bg-card px-1.5 py-0.5 text-[10.5px] text-faint">ESC</span>
        </div>
        <div className="max-h-[344px] overflow-y-auto p-2">
          {results.map((it, idx) => {
            const Icon = it.icon
            return (
              <button
                key={it.key}
                type="button"
                onMouseEnter={() => setActive(idx)}
                onClick={() => choose(it)}
                className={cn(
                  'flex w-full items-center gap-3 rounded-[9px] px-3 py-2.5 text-left text-[14px] transition-colors',
                  idx === active
                    ? 'bg-[var(--accent-soft)] font-semibold text-[var(--accent)]'
                    : 'text-foreground',
                )}
              >
                <Icon className="h-4 w-4 shrink-0" />
                <span className="flex-1">{it.label}</span>
                <span className="font-mono text-[9.5px] uppercase tracking-[0.12em] text-faint">{it.hint}</span>
                {idx === active && <CornerDownLeft className="h-3.5 w-3.5 text-faint" />}
              </button>
            )
          })}
          {results.length === 0 && (
            <div className="px-5 py-8 text-center text-[13px] text-faint">No matches</div>
          )}
        </div>
        <div className="flex gap-4 border-t border-[var(--hair)] px-5 py-2.5 font-mono text-[10.5px] text-faint">
          <span>↑↓ navigate</span>
          <span>↵ open</span>
          <span>esc close</span>
        </div>
      </DialogContent>
    </Dialog>
  )
}
```

## Task 29: Add scroll-reveal animation hook for landing page

**Objective:** Create a reusable hook that uses IntersectionObserver to reveal elements on scroll, matching the `data-reveal` pattern from the landing page prototype.

**Files:**
- Create: `src/lib/useReveal.ts`

**Step 1: Write the hook**

```ts
'use client'

import { useEffect } from 'react'

// IntersectionObserver-based scroll reveal. Add `data-reveal` to any element
// and it will fade+rise into view when scrolled to. Used by the landing page.
export function useReveal() {
  useEffect(() => {
    const els = document.querySelectorAll('[data-reveal]')
    if (!els.length) return

    els.forEach((el) => {
      const htmlEl = el as HTMLElement
      htmlEl.style.opacity = '0'
      htmlEl.style.transform = 'translateY(36px)'
      htmlEl.style.transition = 'opacity 0.85s cubic-bezier(0.2,0.7,0.2,1), transform 0.85s cubic-bezier(0.2,0.7,0.2,1)'
    })

    const io = new IntersectionObserver((entries) => {
      entries.forEach((e) => {
        if (e.isIntersecting) {
          const el = e.target as HTMLElement
          el.style.opacity = '1'
          el.style.transform = 'none'
          io.unobserve(e.target)
        }
      })
    }, { threshold: 0.12, rootMargin: '0px 0px -8% 0px' })

    els.forEach((el) => io.observe(el))
    return () => io.disconnect()
  }, [])
}
```

In the landing page component (Task 18/19), add at the top of the component:

```tsx

  useReveal()
  // ...
}
```

## Risks, Tradeoffs, and Open Questions

### Risks

1. **`next/image` with local PNGs**: The Aureus logo images are small (~12KB). If `next/image` causes issues with unoptimized mode in dev, add `images: { unoptimized: true }` to `next.config.js`.

2. **Font loading performance**: Loading 4 Google Fonts (Newsreader, Spectral, Hanken Grotesk, IBM Plex Mono) adds weight. Consider using `next/font` for optimization, but the prototype uses `@import` in CSS which works fine for now.

3. **Color class purging**: Tailwind's JIT may purge `text-up`, `text-down`, etc. if they're only used via CSS variables. The `colors` extension in `tailwind.config.ts` (Task 3) handles this, but verify that `bg-down`, `text-cool` etc. appear in the built CSS. If purged, add them to `safelist` in tailwind config.

4. **Sidebar width change**: Changing from `w-14` (56px) to `w-[250px]` changes the main content offset. All pages use `md:pl-14` in the layout — this is updated to `md:pl-[250px]` in Task 15. But any page that hardcodes `pl-14` will need updating. Search for `pl-14` across the codebase.

5. **Hardcoded color classes in non-page files**: Some components in `src/components/` may use hardcoded colors (e.g., `analytics/BreakdownChart.tsx`, `dashboard/AllocationChart.tsx`). These should also be updated. Search for all occurrences:
   ```bash
   grep -rn 'text-\[#ff7a59\]\|text-emerald-400\|text-sky-400\|bg-sky-400\|bg-amber-500\|bg-primary' src/components/
   ```

### Tradeoffs

- **No framer-motion**: The prototypes use pure CSS animations, not framer-motion. This keeps the bundle smaller and matches the prototype exactly, but means more complex animations (like shared element transitions) rely on the View Transitions API which is Chrome-only.
- **Landing page as root route**: Making the landing page the root `/` means existing users who bookmarked `/` will see the marketing page. This is intentional — the README-redesign says "route unauthenticated users here instead of straight to /login". Authenticated users navigate to `/dashboard`.

### Open Questions

1. **Should the landing page have its own layout (no sidebar)?** — Yes, it should. The landing page is a public route without the dashboard sidebar. If it's at `src/app/page.tsx` (root), it inherits the root layout which has no sidebar — correct. The dashboard layout at `src/app/(dashboard)/layout.tsx` adds the sidebar only for dashboard routes.

2. **Should we add a pricing section to the landing page?** — The README mentions a "2-tier pricing block (Personal free / Pro $9)" but the prototype `Aureus Landing.dc.html` doesn't include it. Defer for now.

3. **Should the command palette include quick actions (Add position, Add transaction)?** — The prototype includes these. The current `CommandPalette.tsx` only has routes + theme toggle. Adding quick actions requires wiring to the Sheet component. Defer to a follow-up if the Sheet (Task 16) is complete.
